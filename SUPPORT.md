# Support

Use this file to decide where to ask for help.

## Questions

Open a GitHub Discussion if discussions are enabled. Otherwise, open an issue with the `question` label.

Include:

- what you tried
- your operating system
- Python version
- Node version
- backend command
- frontend command
- the exact error message with secrets removed

## Bugs

Use the bug report issue template.

Never include:

- Gmail app passwords
- Groq or Gemini keys
- `FERNET_KEY`
- `.env`
- `KEYS.md`
- raw database files
- private inbox content

## Security

Use [SECURITY.md](SECURITY.md). Do not report vulnerabilities with exploit details in a public issue.
